## 身についたこと
- パスの動的管理: Pathlib を使い、base_path / "Log" のようにフォルダとファイルを結合して、実行環境に依存しない堅牢なパス指定ができるようになった。

- ディレクトリの自動生成: mkdir(parents=True, exist_ok=True) を使い、プログラムが動作する前に必要な環境（ログ用フォルダなど）を自ら準備する「自己完結型」のロジックを習得した。

- 日付を活用したファイル命名: datetime.now().strftime("%Y%m%d") をファイル名に組み込み、上書きを防ぎつつ時系列で履歴を管理する手法を学んだ。

## エラーと解決策
エラー: load_rules 関数が None を返してしまい、その後の仕分け処理が止まった。

解決策: 昨日のバリデーション機能追加時の「インデントのズレ」が原因と特定。return 文が if 文や try のスコープ外に漏れていたのを修正し、関数の出口（戻り値）を確実に確保することで解消した。

## 実務での活用案
- 監査対応・証拠保全: 実行日ごとにログを分けることで、「いつ、どのファイルが、なぜ移動した（あるいは失敗した）のか」を迅速に検索・報告できる体制が整った。

- ツールのクリーンな配置: ログを専用フォルダ（Logs/）に集約することで、メインのディレクトリが汚れるのを防ぎ、ユーザーが使いやすい構成に改善できた。
