# 1. 実装した主要機能
- 設定ファイルの存在チェック: config.json がない場合に、パニックにならず「見つかりません」と伝えて安全に終了するロジックを実装。
- JSON構文エラーの捕捉: カンマ忘れなどの記述ミスを json.JSONDecodeError で検出し、具体的なエラー原因を表示するようにしました。
- 空設定の防止: ファイルはあっても中身が空（{}）の場合に、それ以降の無駄な処理をスキップして終了するガードを追加。

# 2. 解決した技術的課題
- sys.exit() による安全な停止: エラーを検知した際、不完全な状態で処理を続行させず、プログラムを即座に終了させる方法を学びました。
- 堅牢な関数設計: load_rules 関数を「ただ読み込むだけ」から「検証して読み込む」プロフェッショナルな形へ進化させました。

# 3. 学んだプログラミングの鉄則
- 「GIGO（Garbage In, Garbage Out）」の回避: 不正な入力（ゴミ）が入れば不正な結果が出る。それを防ぐために、入り口で厳しくチェックすることの重要性を実践しました。
